A propos de la base de donnée¶

Cette base de donnes (lien) contient 76 attributs, mais toutes les expriences publies se rfrent l'utilisation d'un sous-ensemble de 14 d'entre eux. En particulier, la base de donnes de Cleveland est la seule qui ait t utilise par les chercheurs de ML ce jour. Le champ objectif fait rfrence la prsence d'une maladie cardiaque chez le patient. Il s'agit d'un entier valant de 0 (pas de prsence) 4.

Je vais donc explorer cette base de donn dans un premier temps et ensuite construire un modle de Machine learning qui prdira si un individu est malade(1) ou non(0)

Informations sur les attributs:¶

  1. ge (age)
  2. sexe (sex)
  3. type de douleur thoracique (4 valeurs) (cp)
  4. pression artrielle au repos(trestbps)
  5. cholestoral srique en mg / dl (chol)
  6. glycmie jeun> 120 mg / dl (fbs)
  7. rsultats lectrocardiographiques au repos (valeurs 0,1,2) (restecg)
  8. frquence cardiaque maximale atteinte(thalach)
  9. angine de poitrineinduite par l'exercice (exang)
  10. oldpeak = dpression ST induite par l'exercice par rapport au repos(oldpeak)
  11. pente du segment ST maximal de l'exercice(slope) 12 nombre de vaisseaux principaux (0-3) colors par une fluoroscopie(ca)
  12. thal: 3 = normal; 6 = dfaut fixe; 7 = dfaut rversible '(thal)

# Fonction pour calculer les valeurs manquantes par colonne# Funct 
def missing_values_table(df):
        # Total des valeurs manquantes
        mis_val = df.isnull().sum()
        
        # Pourcentage des valeurs manquantes
        mis_val_percent = 100 * df.isnull().sum() / len(df)
        
        # Faire un tableau avec les rsultats
        mis_val_table = pd.concat([mis_val, mis_val_percent], axis=1)
        
        # Renommez les colonnes
        mis_val_table_ren_columns = mis_val_table.rename(
        columns = {0 : 'Missing Values', 1 : '% of Total Values'})
        
        # Trier le tableau par pourcentage de manquant dcroissant
        mis_val_table_ren_columns = mis_val_table_ren_columns[
            mis_val_table_ren_columns.iloc[:,1] != 0].sort_values(
        '% of Total Values', ascending=False).round(1)
        # Imprimer quelques informations sommaires
        print ("Your selected dataframe has " + str(df.shape[1]) + " columns.\n"      
            "There are " + str(mis_val_table_ren_columns.shape[0]) +
              " columns that have missing values.")
        
        # Renvoie le dataframe avec les informations manquantes
        return mis_val_table_ren_columns

# #############################################################################
# Define a pipeline 
# classifier
pipeline = Pipeline([('mms', MinMaxScaler()),
                     ('reduce_dim', LinearDiscriminantAnalysis()),
                     ('classify', LogisticRegression())])

# uncommenting more parameters will give better exploring power but will
# increase processing time in a combinatorial way
parameters = {'classify__penalty': ['l1', 'l2'],
                 'classify__C': [0.05,0.1, 0.3, 0.6, 0.8, 1.0]}

if __name__ == "__main__":
    # multiprocessing requires the fork to happen in a __main__ protected
    # block

    # find the best parameters for both the feature extraction and the
    # classifier
    grid_search = GridSearchCV(pipeline, parameters, n_jobs=-1, verbose=1)

    print("Performing grid search...")
    print("pipeline:", [name for name, _ in pipeline.steps])
    print("parameters:")
    pprint(parameters)
    t0 = time()
    grid_search.fit(X_train, y_train)
    print("done in %0.3fs" % (time() - t0))
    print()

    print("Best score: %0.3f" % grid_search.best_score_)
    print("Best parameters set:")
    best_parameters = grid_search.best_estimator_.get_params()
    for param_name in sorted(parameters.keys()):
        print("\t%s: %r" % (param_name, best_parameters[param_name]))
